中间表的关系，本质上是靠**两个外键**建立的。

拿你项目里的页面和资源来说：

```text
Page 页面
Resource 资源
```

它们是多对多关系：

```text
一个 Page 可以引用多个 Resource
一个 Resource 也可以被多个 Page 复用
```

所以中间加一张表：

```text
PageResourceRelation
```

这张表里存：

```text
pageId
resourceId
```

它的意思是：

> 哪个页面引用了哪个资源。

**在 Prisma 里是这样建立的**

你项目里的结构大概是：

```prisma
model Page {
  id                String                 @id @default(uuid()) @db.Uuid
  resourceRelations PageResourceRelation[]
}

model Resource {
  id                String                 @id @default(uuid()) @db.Uuid
  pageRelations     PageResourceRelation[]
}

model PageResourceRelation {
  id         String   @id @default(uuid()) @db.Uuid

  pageId     String   @db.Uuid
  resourceId String   @db.Uuid

  page       Page     @relation(fields: [pageId], references: [id])
  resource   Resource @relation(fields: [resourceId], references: [id])

  @@unique([pageId, resourceId])
  @@index([pageId])
  @@index([resourceId])
}
```

关键是这两句：

```prisma
page Page @relation(fields: [pageId], references: [id])
```

意思是：

> `PageResourceRelation.pageId` 指向 `Page.id`

这句：

```prisma
resource Resource @relation(fields: [resourceId], references: [id])
```

意思是：

> `PageResourceRelation.resourceId` 指向 `Resource.id`

所以中间表实际长这样：

```text
PageResourceRelation
------------------------------------------------
id          pageId        resourceId
rel_001     page_001      res_001
rel_002     page_001      res_002
rel_003     page_002      res_001
```

这表示：

```text
page_001 引用了 res_001
page_001 引用了 res_002
page_002 也引用了 res_001
```

所以：

```text
Page page_001 -> Resource res_001、res_002
Resource res_001 -> Page page_001、page_002
```

**为什么要有反向字段**

在 `Page` 里：

```prisma
resourceRelations PageResourceRelation[]
```

表示：

> 一个页面可以有多条资源关系。

在 `Resource` 里：

```prisma
pageRelations PageResourceRelation[]
```

表示：

> 一个资源可以被多个页面引用。

这样你就可以查：

```ts
const page = await prisma.page.findUnique({
  where: { id: pageId },
  include: {
    resourceRelations: {
      include: {
        resource: true,
      },
    },
  },
})
```

也可以反查：

```ts
const resource = await prisma.resource.findUnique({
  where: { id: resourceId },
  include: {
    pageRelations: {
      include: {
        page: true,
      },
    },
  },
})
```

**为什么要 `@@unique([pageId, resourceId])`**

这个是为了防止重复关系。

否则可能出现：

```text
page_001 -> res_001
page_001 -> res_001
page_001 -> res_001
```

同一个页面重复绑定同一个资源，没有意义。

所以加：

```prisma
@@unique([pageId, resourceId])
```

表示：

> 同一个 pageId + resourceId 组合只能出现一次。

这也是为什么代码里可以用：

```ts
upsert({
  where: {
    pageId_resourceId: {
      pageId,
      resourceId,
    },
  },
})
```

因为这个组合是唯一的，可以准确定位一条关系。

**面试可以这样说**

> 中间表是用来表达多对多关系的。比如一个页面可以引用多个资源，一个资源也可以被多个页面复用，所以不能只在 Page 表或者 Resource 表里放一个字段。我们用 `PageResourceRelation` 表存 `pageId` 和 `resourceId` 两个外键，分别关联到 `Page.id` 和 `Resource.id`。同时加 `pageId + resourceId` 联合唯一约束，避免重复绑定。查询页面详情时，通过 include 中间表，再 include resource，就可以拿到页面引用的资源列表。

一句话记：

> 中间表 = 两个外键 + 两边反向关系 + 联合唯一约束。
