`SlidePublishRecord` 是**一次课件发布的总记录**。

它表示：

```text
某个课件在某个时间发布了一次，生成了版本 v1 / v2 / v3
```

它管的是“整套课件”的发布状态。

比如字段：

```text
slideId            属于哪个课件
version            第几个发布版本
status             publishing / success / failed
slideSnapshot      发布时课件整体结构
manifest           播放端入口清单
manifestOssPath    manifest 地址
packageOssPath     离线包地址
errorMessage       发布失败原因
```

可以理解为：

```text
SlidePublishRecord = 发布版本总表
```

---

`PagePublishSnapshot` 是**这次发布里每一页的快照**。

它表示：

```text
在某次发布版本中，第 1 页、第 2 页、第 3 页分别长什么样
```

它管的是“单页”的固化数据。

比如字段：

```text
publishRecordId     属于哪次发布
pageId              原始编辑态页面 ID
pageType            页面类型
sortIndex           页面顺序
pageSnapshot        发布时这一页的 JSON
resources           这一页引用的资源快照
screenshotOssPath   这一页缩略图地址
```

可以理解为：

```text
PagePublishSnapshot = 发布版本里的页面明细表
```

---

举个例子。

一个课件有 3 页，你发布一次，生成 `v1`。

那么数据库里会有：

```text
SlidePublishRecord
  id: record-v1
  slideId: slide-001
  version: 1
  status: success
```

同时有 3 条页面快照：

```text
PagePublishSnapshot
  publishRecordId: record-v1
  pageId: page-001
  pageSnapshot: 第 1 页发布时的 JSON
  screenshotOssPath: 第 1 页缩略图

PagePublishSnapshot
  publishRecordId: record-v1
  pageId: page-002
  pageSnapshot: 第 2 页发布时的 JSON
  screenshotOssPath: 第 2 页缩略图

PagePublishSnapshot
  publishRecordId: record-v1
  pageId: page-003
  pageSnapshot: 第 3 页发布时的 JSON
  screenshotOssPath: 第 3 页缩略图
```

如果后来你更新发布，生成 `v2`，又会新增：

```text
SlidePublishRecord record-v2
PagePublishSnapshot page-001 in v2
PagePublishSnapshot page-002 in v2
PagePublishSnapshot page-003 in v2
```

旧的 `v1` 不动。

一句话：

**`SlidePublishRecord` 记录“发布了一版课件”；`PagePublishSnapshot` 记录“这一版课件里的每一页当时是什么内容”。**
