# Slides Engine — 高频面试问题与参考答案（贴合本项目）

> 适用于高级前端 / 全栈 / 架构岗；口述时优先 **结论 → 本项目代码落点 → 权衡**，忌纯背诵八股。

---

## 架构与设计

### 1. 你们课件编辑器和市面上 low-code（如 Formily、Designable）是什么关系？内核边界在哪里？

**答**：整体范式接近 **Designable**：`Engine / Workspace / Operation / TreeNode` 驱动画布，`JSON Schema` 驱动属性表单（Formily 系）。本项目在通用内核外包了一层 **教学课件物料**（`editor/RegistryBehaviors`：Root、Card、Shape、Game…）。

**内核边界**：`packages/core` 只关心「树怎么改、怎么选、怎么拖、怎么撤销」，不关心具体课件组件；业务组件注册、COS、预览 URL、课堂协议在 **editor / task / bridge**。

---

### 2. 为什么是 `Engine → Workspace → Operation`？若加「协作编辑」挂在哪里？

**答**：**Engine** 全局事件与快捷键；**Workspace** 对应「一页课件」——独立视口、独立历史；**Operation** 承载当前页的 **Tree + Selection + MoveHelper + TransformHelper + AnimateEngine**。

协作编辑可在 **Workspace 之上**加同步层：要么把每次 mutation 打成 **OperationLog** 广播（保留现有快照撤销），要么引入 CRDT 文档层替换局部 TreeNode 写入；不宜把协作逻辑塞进单个 React 组件。

---

### 3. `Workspace` 为什么要两个 `Viewport`？`moveInsertionType` 不同解决什么？

**答**：同一页既要 **所见即所得画布**，又要 **大纲树**。二者共用 `Operation` 与 `MoveHelper`，但配置不同：

- 主画布：`moveInsertionType: 'all'`，`moveSensitive: true`，支持细粒度插入。
- 大纲：`moveInsertionType: 'block'`，`moveSensitive: false`，强制块状层级调整，避免大纲里出现画布级精细插入。

同时 `nodeIdAttrName` 与 `outlineNodeIdAttrName` 区分 DOM 映射。

---

### 4. `GlobalRegistry` + `createBehavior` 扩展新物料要改哪些层？哪些不能动？

**答**：通常新增 **Behavior（selector + designerProps + propsSchema）**、编辑态组件、运行态渲染（`@slide/render-components` 等）、必要时 Setter。**尽量不要改** `Engine`/`TreeNode` 事件总线、`History` 语义；除非新物料需要全新 mutation 类型，再在 core 增加事件并在 `TreeNode` 处理。

---

### 5. 树状态用 `@slides/reactive` 而非 Redux，理由与代价？

**答**：画布是 **高频、深层嵌套、局部字段变更**，细粒度 observable 减少无效渲染；Redux 更适合可追溯的业务流水。

**代价**：撤销不靠 Redux time-travel，而靠 **`History` 快照**；调试「某一 prop 谁改的」要依赖事件或自定义日志。

---

## 数据与协议

### 6. 课件持久化最小结构？`pageType`、`serialize()` 语义？

**答**：最小单位是 **一页一棵组件树**（`ITreeNode`：`componentName`、`props`、`children`、`id`…）。`Workspace.serialize()` 产出含 **`pageType`、`pageInfo`（即树的序列化结果）** 等，便于预览端按页还原。

`pageType` 表达页类型（如普通 / 游戏 / 视频），与 `task` 里 `PageType` 枚举等业务概念对齐。

---

### 7. 编辑端到预览的数据契约谁定义？版本升级如何兼容？

**答**：契约体现为 **序列化字段**：父窗口 `JSON.stringify({ slideId, pageId, pages, fileList })`，子窗口 `getSlideData` 解析后 `dispatch.page.updatePage`。本质是前后端 + 双端约定。

兼容：**增加 `schemaVersion`**；预览端「读旧写新」——未知字段忽略，缺省字段给默认值；保存时可选迁移脚本。

---

### 8. `task` 里 `globalProps.fileList` 与 CDN 如何影响渲染？失败如何降级？

**答**：`globalProps` 承载 **资源列表、首帧样式**（如 `opacity`/`visibility` 防闪烁）等；`globalConfig.resourceData.remote.cdnPathList` 指向 CDN。

降级：**资源 404** 用占位图/占位文案；**CDN 不可用** 可切换备用域名或本地缓存列表（需产品策略）；解析失败应 **catch + Toast**，避免白屏。

---

## 交互与算法

### 9. 拖拽如何定插入位置？`ClosestPosition` 如何参与 drop？

**答**：`MoveHelper` 根据指针与候选节点矩形计算 **最近关系**：Before / After / Inner 等（`ClosestPosition`）。结合 **activeViewport** 与规则（`allowAppend`、`moveInsertionType`）决定能否放置；最终触发 **`DragNodeEvent` / `DropNodeEvent`** 一类 mutation，由 `TreeNode` 改父子关系。

---

### 10. 源节点与画布实例区别？拖拽取消如何回滚？

**答**：物料栏多为 **`isSourceNode`**，进画布时常 **`clone(parent)`** 成实例；已在树上的节点移动则 **remove + 改 parent**。

取消：**若未 Drop 成功**，引擎应清除 `dragNodes`、恢复 hover；若已部分改树，依赖 **History**：关键 drop 结束 `snapshot`，取消中途不产生快照则可依赖最后一次一致状态（具体以当前 Driver 实现为准，口述时强调「要么事务式变更要么入栈」）。

---

### 11. 为何撤销用快照不用命令？何时改命令模式？

**答**：快照与 **现有「整树 JSON 持久化」**一致，实现快、行为直观；课件规模可控时序列化成本可接受。

改为命令模式：**节点规模极大**、序列化成为瓶颈，或需要 **跨会话协作合并** 时，用操作日志 + 逆操作更划算。

---

## 跨端与安全

### 12. iframe 预览为何用 post-me？`remoteOrigin` 不用 `*` 的意义？

**答**：post-me 提供 **Promise 化握手**（`ParentHandshake` / `ChildHandshake`），比裸 `postMessage` 好组合生命周期。

**显式 `remoteOrigin`**：限制消息只对可信子域生效，降低恶意页面冒充 iframe 窃取或注入数据的风险（仍需防 XSS）。

---

### 13. 预览域与编辑域不同站，鉴权与传递如何设计？

**答**：postMessage 传 **短期 token** 或 **slideId + 签名**，子域用 token **换完整 JSON**（同源 API）；或对 payload **签名/HMAC**，子窗口验签后再解析。避免把敏感密钥放在前端，敏感逻辑放网关。

---

### 14. `PostMessageClient` 里 microApp 与 BroadcastChannel 分支？

**答**：**BroadcastChannel**：本地/预览模拟、类课堂「广播」多窗口同步。**microApp**：真实 **微前端宿主**，走 `forceDispatch` / `addDataListener` 与主应用通信。同一渲染核适配两种运行时。

---

## 性能与质量

### 15. `snapshot` 里 `requestIdle` 导致撤销粒度变化，如何取舍？

**答**：换取 **主线程空闲时批处理**，减少卡顿。若产品要求「每一步必 undo」，可对 **离散操作**（drop 结束、blur）**同步 push**，连续输入仍 idle 合并。

---

### 16. History `maxSize` 100 的依据？线上如何观测调参？

**答**：平衡 **内存与可回溯步数**；教学场景多数不需要上千步。

观测：监控 **serialize 耗时**、**内存**、用户误操作投诉；大课件可调小到 50 或改为分层快照。

---

### 17. 如何给「整树 serialize」做 profiling 与优化路线？

**答**：Chrome Performance + 自定义标记计量 `serialize()`；对大节点树 **采样对比 JSON.stringify vs 结构化克隆**。

路线：**Worker 序列化**、**子树增量快照**、**关键帧 + 操作日志**混合。

---

## 业务与扩展

### 18. 嵌入游戏与 `bridge` 职责？加载失败编辑器如何提示？

**答**：`bridge`（如 mcc-player、cocos-game-player）承载 **宿主页 ↔ 游戏 iframe ↔ 原生** 的消息协议。

编辑器侧：`GameBehavior` / Modal 监听 **超时与 error 事件**，Toast + 允许重新选择游戏资源；不要把逻辑散落在多处 postMessage。

---

### 19. `AnimateEngine` 挂在 `Operation` 利弊？动画要持久化怎么改？

**答**：**利**：动效与当前页操作上下文近，便于联动选区与预览。**弊**：core 与 `@slides/animate` 耦合加重。

持久化：在 **`ITreeNode.props` 或独立 `animations` 字段**存配置，`serialize`/`from` 纳入；`AnimateEngine` 从树初始化，而非仅存内存。

---

### 20. 任务系统与画布节点绑定键？如何保证稳定？

**答**：任务模型含 **`elementId`**（README 中 `ITask`），对应画布节点 **稳定 id**。

保证：**生成节点即固化 id**，禁止随意重生成；复制页时 **映射表**更新引用；保存前校验任务引用的 id 仍存在。

## AI / 智能化（扩展）

### 21. 大模型生成课件 JSON 如何校验与安全？

**答**：**JSON.parse + JSON Schema / Zod + componentName 白名单**；禁止 `eval`。外链仅 **HTTPS + 域名白名单**；敏感字段二次确认。

---

### 22. AI 排版结果如何接入 History？

**答**：AI 仅调用现有 API（`setProps`、`append`…），末尾 **`operation.snapshot()`** 或批量变更包一层「事务」后 **一次 push**，用户 **Ctrl+Z** 整块撤销。

---

## 工程化

### 23. pnpm `workspace:^` 冲突如何排查？

**答**：`pnpm why <pkg>`、`pnpm list -r`；检查 **多版本并存**、peer 不满足；锁定 **`pnpm-lock.yaml`**；必要时 `overrides` 统一版本。

---

### 24. editor 目录与 package 名 `play` 不一致如何治理？

**答**：**迁移优先级**：更新 `package.json` `name`、文档、CI、脚本路径；或保留别名但在 README **显著说明**「目录 editor，包名 play」。长期应统一以免 onboarding 成本。
