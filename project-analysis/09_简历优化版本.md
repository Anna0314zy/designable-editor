# Slides Engine — 简历优化版本

> 以下为**项目真实技术点**提炼，请按你个人职责取舍增删，避免夸大未参与部分。

---

## 一句话项目描述

教育场景下的 **交互式课件 DSL 编辑引擎**：基于 **可观察组件树 + Behavior 注册扩展**，支撑多页课件设计、跨 iframe 预览协议、课堂播放与任务编排；内核层参考 Designable 范式并沉淀教学侧物料（图形、富文本、音视频、游戏嵌入等）。

---

## 三条 Bullet（高级前端 / 架构向）

1. **课件低代码内核**：基于 `Engine / Workspace / Operation` 抽象实现画布树、双视口（画布+大纲）、选区与拖拽插入算法（`MoveHelper` + `ClosestPosition`）；使用 **History 序列化快照 + requestIdle 合并** 优化撤销体验与主线程占用。

2. **可扩展物料体系**：通过 `GlobalRegistry` + `createBehavior` 将 **JSON Schema 属性面板、拖拽规则、国际化** 与组件绑定；编辑应用集中 `RegistryBehaviors` 注册 Root/Card/Game 等业务组件，实现新增物料 **零改引擎核心**。

3. **跨端渲染与集成**：设计并实现编辑端与预览端的 **post-me 握手 + 显式 remoteOrigin** 数据传输；了解 **微前端宿主（microApp）与 BroadcastChannel** 双路径消息适配，支撑课件在不同部署形态下的一致播放。

---

## 两条 Bullet（偏工程化 / 质量）

- 推动 **Workspace 级序列化契约** 与预览 Store（Rematch）对齐，梳理 `pageType`、全局 `globalProps`（CDN、资源列表）对渲染的影响边界。
- 使用 **PWA + 字体按需加载** 改善编辑器资源更新体验；通过 Modal `destroyOnClose` 管理预览 iframe 生命周期，降低常驻内存。
