# Slides Engine — AI 化扩展方向

> **声明**：当前仓库主体为可视化编辑与播放，**不含** LLM/RAG 实现。下文为**基于现有架构的演进设计**，满足你对「AI Native 扩展」的分析维度。

---

## 1. 与本项目强相关的 AI 场景

| 场景 | 输入 | 输出 | 接入点 |
|------|------|------|--------|
| 自然语言生成一页课件 | 用户描述 | `ITreeNode` 子树 | 生成后 `operation.tree` 合并 + `snapshot` |
| 组件解释 / 改稿建议 | 选中节点 JSON | 文案或 props 建议 | 侧栏 Panel，写回 `setProps` |
| 配图 / 配图槽位 | 主题、页大纲 | 图片 URL + alt | `Img` 节点 props |
| 课堂话术推荐 | 当前任务类型 + 页内容 | 台词文案 | 与 `task` 的 `words` 字段联动 |
| 可访问性 / 合规检查 | 树 JSON | 问题列表 | CI 或保存前校验 |

---

## 2. 模型选型与部署（若引入 LLM）

- **公有 API**：低延迟迭代、适合 MVP；注意课件内容 **隐私与合规**（未成年人场景尤甚）。
- **私有化**：教育政企常见需求；部署在内网，通过 **网关统一鉴权**，前端仅持有短期 token。
- **多模型兼容**：抽象 **`LLMProvider` 接口**（`complete`、`stream`、`embed`），配置驱动切换厂商；与现有 **`GlobalRegistry`** 思想一致——插件式适配器。

**本项目切入**：不动内核，在 **editor 壳层** 增加 `AiAssistantPanel`，通过引擎 API 读写树。

---

## 3. Prompt 与上下文管理

- **系统 Prompt**：固定输出 **合法 JSON Schema 子集**，限定 `componentName` 白名单（与 `RegistryBehaviors` 注册名一致）。
- **上下文**：当前选中节点路径、`serialize()` 当前页摘要（可裁剪 children 深度）、品牌色等 **globalProps**。
- **记忆**：会话级 **短期记忆**（React state / sessionStorage）；跨会话用服务端 **课件版本** 存储，而非让模型记全文。

**关键**：模型输出必须经过 **`JSON.parse` + Schema 校验 + 组件白名单**，再调用 `TreeNode` API，禁止 `eval`。

---

## 4. RAG（若接入教案 / 教材知识）

当前项目无向量库。**若扩展**：

- **数据源**：校本教案 PDF、课标段落；切块时保留 **章节标题** 元数据。
- **分块**：按段落 + 重叠窗口；图表单独 OCR 管道。
- **召回**：向量检索 + **关键词过滤（年级、学科）**；排序可加 cross-encoder 重排。
- **落地**：检索结果仅作为 Prompt **引用片段**，生成课件仍需走 Schema 校验。

**与 slides-engine 结合**：检索到的知识点映射为 **预设节点模板**（如「知识点卡片」Behavior）。

---

## 5. Agent / 工具调用（编排）

若要做「多步改课件」：

- **工具**：`get_current_page_json`、`patch_node_props`、`add_node`、`run_preview_validation`。
- **状态**：维护 **AgentState**（独立于 Engine），每步工具调用后同步到 `Operation` 并 `snapshot`，保证用户可撤销。
- **子图**：复杂任务拆成「布局子 Agent」「配图子 Agent」，由编排层合并结果，避免单 Prompt 过长。

---

## 6. AI 链路性能与成本

- **推理耗时**：流式输出 UI；大纲先行、节点逐个挂载。
- **并发限流**：网关 + 前端队列；同一 `slideId` 编辑会话串行化写树。
- **缓存**：相同「章节 ID + 模板版本」的生成结果可缓存 **结构化 JSON**，不是缓存原文。
- **Token 优化**：传给模型的树做 **剪枝**（仅保留 componentName + 关键 props）；大段正文用占位符引用。

---

## 7. 幻觉与安全

- **幻觉**：强制 Schema；非法结构丢弃并重试（repair prompt）；关键数字 **人工确认** 才可入库。
- **内容风控**：文本与图片 URL 走 **合规 API**；外链图片域名白名单。
- **可解释性**：侧栏展示「依据的教案段落 ID / 引用片段」，便于教师审核。

---

## 8. AI 工程化

- **模型版本**：配置表记录 `model_id`；课件 JSON 存 `ai_meta.model_version`。
- **灰度**：按租户 / 用户尾号开放 AI 面板。
- **Prompt 版本**：与 `model_version` 绑定；回归测试用 **固定 seed + 快照对比树 JSON diff**。

---

## 9. 从「传统编辑项目」演进到 AI Native（本仓库适用）

**现状定位**：可视化内核 + 教学业务插件，无模型推理链路 —— 属于 **传统智能尚未内置** 的系统。

**重构分层（推荐顺序）**：

1. **不改内核，先加「提议层」**：所有 AI 输出经 **校验器** 转成合法 `ITreeNode` patch，通过现有 `setProps` / `append` API 应用，并走 **`snapshot`**，保证可撤销。
2. **引入领域护栏**：课件保存 API 增加 **`ai_provenance`**（是否含 AI 生成、模型版本、人工确认标记），满足校内合规审计。
3. **记忆与上下文外挂**：用独立 **Copilot Service** 维护对话与检索结果索引；内核只暴露「当前选中子树摘要」查询接口，避免把会话状态塞进 `Operation`。
4. **Agent 化**：当单轮 Prompt 不足时，再引入工具调用循环；工具仅限 **白名单操作**（禁止任意 HTTP）。
5. **最终形态**：内核仍是 **确定性 DSL 引擎**；AI 成为 **编译器前端**（自然语言 → 树），而非运行时替换渲染 —— 这与「AI Native」中 **可控、可审计、可回滚** 的教育产品要求一致。

---

## 10. 小结（面试一句话）

> Slides Engine 的 AI 化应 **外挂于引擎**：LLM 只负责提案，**TreeNode + History** 仍是唯一真相；协议层补齐 **Schema 校验与行为白名单**，再配上私有化网关与审计字段，即可演进为教育场景可控的 **Copilot**，而非替换内核的 black box。
