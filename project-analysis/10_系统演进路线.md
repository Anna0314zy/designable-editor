# Slides Engine — 系统演进路线

面向 **6～18 个月** 可落地的分期规划，与当前代码缺口对齐。

---

## Phase 0 — 基线治理（1～4 周）

| 项 | 动作 | 价值 |
|----|------|------|
| 命名一致 | `editor` package `play` 命名、根 README 与 pnpm workspace 对齐 | 降低入职与 CI 配置错误 |
| 日志 | 生产剥离 `console.log`（animate、PostMessageClient） | 性能与安全 |
| 协议版本 | `serialize` 根增加 `schemaVersion` | 兼容升级 |
| 错误边界 | 预览 `getSlideData` 失败 UI + 重试 | 课堂稳定性 |

---

## Phase 1 — 性能与规模（1～3 月）

1. **Profiling 整页 serialize**：大课件瓶颈验证；必要时 Worker 化或子树增量快照。
2. **大纲虚拟化**：节点数阈值以上启用列表虚拟滚动。
3. **拖拽 rAF**：若 MoveHelper 热点确认，合并 move 事件计算。

---

## Phase 2 — 契约与测试（并行）

- **Golden JSON**：核心物料的 `serialize` 样例快照测试。
- **跨端契约测试**：模拟 ParentHandshake 最小用例（Playwright / Vitest + iframe）。
- **Schema 校验**：保存与预览入口统一 Zod 校验，防止脏数据落库。

---

## Phase 3 — AI Copilot（外挂式）

- 见 `07_AI化扩展方向.md`：侧栏面板 + LLM Provider 抽象 + 输出 **Schema 校验后** 写树。
- 一期可做 **文案辅助 / 单组件 props 建议**，避免一上来全页生成。

---

## Phase 4 — 协作与实时（可选，依赖产品）

- **可选 OT/CRDT**：仅在「多教师同编」需求明确时投入；先在 TreeNode mutation 层抽象 **OperationLog**。
- **会话级锁**：轻量方案为页面级编辑锁 + 心跳。

---

## Phase 5 — 商业化与多租户（若适用）

- **租户级 CDN、字体、组件白名单** 写入 `globalConfig`。
- **审计日志**：谁改了哪一节点（需服务端配合）。

---

## 风险与依赖

- **游戏 / 原生桥**：依赖客户端版本节奏，需 **协议版本** 与降级页面。
- **微前端宿主**：与基建团队对齐 microApp API 变更。

---
